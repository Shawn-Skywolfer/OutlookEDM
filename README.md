# Outlook EDM 制作专家

一个面向 Outlook 邮件场景的浏览器端 EDM 制作工具。它可以将多张图片按顺序组合成邮件，也可以先把一张完整的长图切割成多个切片，再生成包含内嵌图片的 `.eml` 邮件文件。

项目采用单文件前端实现，不需要构建步骤，也不会将上传的图片发送到服务器；图片读取、切片和邮件生成均在当前浏览器中完成。

## 主要功能

- **中英双语界面 / Bilingual interface**
  - 点击页面右上角的 `中文 / EN` 可随时切换界面语言。
  - Use the `中文 / EN` control in the upper-right corner to switch languages at any time.
  - 语言偏好会保存在浏览器中，并在下次打开时自动恢复。
  - Your language preference is saved in the browser and restored on your next visit.

- **上传切片模式**
  - 一次上传多张图片。
  - 根据文件名中的数字自动进行初始排序。
  - 在预览区拖拽图片以调整最终顺序。
  - 使用厘米设置 EDM 在邮件中的显示宽度。
  - 超出预览区域宽度时，可通过横向滚动条查看完整内容。

- **长图切割模式**
  - 上传一张完整设计长图。
  - 像 Photoshop 切片工具一样纵向拖拽创建横贯图片宽度的切片。
  - 支持切片边缘吸附、相邻切片边界联动调整和尺寸提示。
  - 支持删除切片、撤销和重做。
  - 将切割结果转换为可继续排序、预览和导出的图片切片。

- **EML 邮件生成**
  - 设置邮件主题。
  - 将图片作为 CID 内嵌资源写入邮件。
  - 生成可由 Outlook 打开的 `.eml` 文件。

## 运行要求

建议使用较新版本的桌面浏览器：

- Microsoft Edge
- Google Chrome

页面启动时需要联网加载以下前端依赖：

- Tailwind CSS
- React 18
- React DOM 18
- Babel Standalone

> 上传的图片只在浏览器本地处理，但首次打开页面时需要能够访问项目中配置的 CDN。

## 启动项目

### 方法一：直接打开

直接双击 `index.html`，或将其拖入浏览器窗口。

这种方式适合快速使用。如果浏览器限制本地文件运行脚本，请使用下面的本地服务器方式。

### 方法二：使用 Python 本地服务器（推荐）

在项目根目录运行：

```bash
python3 -m http.server 8080
```

然后在浏览器中访问：

```text
http://localhost:8080
```

### 方法三：使用 Node.js 本地服务器

如果已经安装 Node.js，可以运行：

```bash
npx serve .
```

按照终端输出的地址打开页面即可。

## 使用方法

应用提供两种素材处理方式。无论使用哪一种方式，最终都会进入“上传切片模式”完成顺序确认、宽度设置和 EML 导出。

---

## 方式一：直接上传已经切好的图片

适用于设计稿已经被切成多张连续图片的情况。

### 1. 进入上传切片模式

打开页面后，点击顶部的 **“上传切片模式”**。

### 2. 设置邮件主题

在左侧 **“邮件主题”** 输入框中填写邮件标题。

该内容会同时用于：

- 生成邮件的主题；
- 下载文件的文件名。

### 3. 设置图片显示宽度

在 **“尺寸设置”** 中输入图片显示宽度，单位为厘米（cm）。默认值为 `15` cm。

这里设置的是图片在最终 Outlook 邮件中的显示宽度，不会改变原始图片文件。图片高度会根据原图比例自动计算。

### 4. 上传图片切片

点击 **“上传多张切片文件”** 区域，一次选择一张或多张图片。

建议使用带连续数字的文件名，例如：

```text
slice_01.jpg
slice_02.jpg
slice_03.jpg
```

工具会提取文件名中的第一个数字进行初始排序。没有数字的文件会排在带数字文件之后。

### 5. 检查并调整图片顺序

上传后，右侧会显示 Outlook 发送预览：

1. 按住某张图片；
2. 将其拖到目标位置；
3. 松开鼠标完成排序。

预览中的排列顺序就是最终邮件中的图片顺序。

当设置的图片宽度超过预览框时，预览区底部会出现横向滚动条，可左右滚动查看完整图片。

### 6. 生成 EML 邮件

确认主题、宽度和顺序后，点击右上角 **“生成 EML 邮件”**。

浏览器会下载一个以邮件主题命名的 `.eml` 文件。

---

## 方式二：上传长图后进行切割

适用于只有一张完整 EDM 长图、需要在工具中创建切片的情况。

### 1. 进入长图切割模式

点击顶部的 **“长图切割模式”**。

### 2. 上传长图

在左侧 **“上传素材”** 区域选择一张图片。长图会显示在右侧切片编辑器中。

当前模式一次只处理一张长图。重新上传会替换当前长图。

### 3. 创建切片

确认工具箱中选中的是 **“切片工具”**，然后在图片的未切割区域操作：

1. 将鼠标移动到希望开始切片的位置；
2. 按住鼠标左键并向上或向下拖动；
3. 松开鼠标完成切片。

创建过程中：

- 切片会始终横贯图片的完整宽度；
- 水平参考线会显示当前位置；
- 尺寸浮层会显示切片的原始像素宽度和高度；
- 靠近已有切片边界时会自动吸附；
- 拖动范围会被限制在相邻切片或图片边缘之间；
- 不允许新切片与已有切片重叠。

也可以从图片左右两侧的灰色区域开始拖拽，纵向位置仍会投影到图片范围内。

> 必须在尚未被切片占用的纵向区域开始创建新切片。如果起点位于已有切片内部，本次绘制不会生效。

### 4. 调整切片边界

选择 **“切片工具”** 后，将鼠标移动到切片的上边缘或下边缘：

- 命中边界时，参考线会高亮，光标会变为纵向调整光标；
- 按住并上下拖动即可修改边界；
- 两个切片相邻时，拖动公共边界会同时调整上下两个切片；
- 独立边缘不会越过相邻切片；
- 工具会保留切片最小高度，并继续应用边界吸附规则；
- 即使鼠标移出编辑区域，当前拖动仍可继续，松开后完成调整。

### 5. 选择切片

- 单击切片：选中当前切片；
- 按住 `Shift` 单击：将切片加入选择或从选择中移除；
- 单击图片外的编辑区域：清除当前选择。

切片左上角的编号表示其从上到下的顺序。

### 6. 删除切片

1. 点击工具箱中的 **“删除切片”**；
2. 将鼠标移动到需要删除的切片；
3. 单击切片完成删除。

删除模式下切片会显示为红色提示。

### 7. 撤销和重做

工具箱提供：

- **撤销**：恢复上一步创建、删除或边界调整；
- **重做**：重新执行刚刚撤销的操作。

执行新的编辑操作后，原有重做记录会被清空。

### 8. 生成切片并进入邮件预览

完成切割后，点击页面右上角绿色的 **“制作 EDM”** 按钮，将当前切片生成图片。

工具会：

1. 按切片从上到下的顺序裁切长图；
2. 将每个切片转换为 JPEG 图片；
3. 自动切换到 **“上传切片模式”**；
4. 在 Outlook 发送预览中显示结果。

之后可以继续：

- 拖拽调整切片顺序；
- 设置邮件显示宽度；
- 修改邮件主题；
- 点击 **“生成 EML 邮件”** 完成导出。

## 在 Outlook 中使用生成的邮件

1. 找到浏览器下载的 `.eml` 文件；
2. 使用 Microsoft Outlook 打开该文件；
3. 检查邮件主题、图片显示效果和切片顺序；
4. 根据需要补充收件人、正文或其他内容；
5. 发送邮件。

建议正式发送前，先发送一封测试邮件到不同尺寸的桌面和移动设备，检查：

- 图片是否完整加载；
- 切片之间是否存在缝隙；
- 邮件宽度是否合适；
- Outlook 不同版本中的显示效果是否一致。

## 数据和输出说明

- 上传图片通过浏览器的 `FileReader` 在本地读取，不会由本项目主动上传到服务器。
- 长图切片通过浏览器 Canvas 完成。
- 长图生成的切片使用 JPEG 格式，质量参数为 `0.9`。
- EML 中的图片通过 CID 作为邮件内嵌资源引用，不依赖远程图片地址。
- 邮件正文会在图片表格前保留三行空白。

## 使用建议

### 图片命名

直接上传切片时，推荐使用补零编号：

```text
edm_01.jpg
edm_02.jpg
edm_03.jpg
```

这样文件管理和上传后的初始顺序更清晰。

### 切片位置

尽量在以下位置切割：

- 纯色背景；
- 留白区域；
- 模块之间的自然分隔处。

避免从文字、按钮、人物面部或复杂渐变中间切开，以减少 Outlook 渲染时切片接缝带来的视觉影响。

### 邮件宽度

`15 cm` 可作为常规桌面邮件的起始值。实际效果与 Outlook 版本、系统缩放和收件设备有关，请以测试邮件为准。

### 图片体积

图片数量多、尺寸大时，生成的 EML 文件也会明显增大。建议在不影响清晰度的前提下，先对素材进行适当压缩。

## 常见问题

### 页面一直停留在加载界面

请检查是否可以访问页面依赖的 CDN，并确认浏览器没有拦截脚本。也可以打开浏览器开发者工具查看 Console 和 Network 信息。

### 上传后图片顺序不正确

工具根据文件名中的第一个数字进行初始排序。请检查文件名编号，或直接在右侧预览中拖拽调整顺序。

### 无法在某个位置创建切片

新切片必须从未被已有切片占用的纵向区域开始，并且不能与已有切片重叠。可以先调整或删除附近切片，再重新绘制。

### 切片边界无法继续拖动

边界受图片范围、相邻切片位置和最小切片高度限制。这是为了避免切片越界、重叠或高度变为零。

### 预览图片超过预览框

当目标显示宽度超过预览区域时，请使用预览区底部的横向滚动条查看完整内容。这不会影响最终导出的邮件宽度。

### 点击“生成 EML 邮件”没有反应

请确认已经上传或生成至少一张切片图片，同时检查浏览器是否阻止了文件下载。
